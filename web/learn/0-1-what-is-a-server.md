# What even is a server? (start here)

Before we touch a single button, let's make sure the words make sense. If you've never run a server before, this lesson is your foundation — five minutes of plain English so nothing later feels like jargon. If you already know this stuff cold, skip ahead with a clear conscience.

## A server is just a computer

That's the whole secret. A **server** is an ordinary computer — same parts as a laptop: a processor, memory, a disk. The only real difference is its *job*. Instead of sitting on a desk running one person's apps, a server sits somewhere quiet (a cupboard, a data centre, a rented box in the cloud) and **runs things other computers connect to**: a website, a game, a file store, a backup.

When people say "the server's down," they just mean "that always-on computer that hosts the thing isn't responding right now." Nothing mystical.

## Linux is what most servers run

Your laptop probably runs Windows or macOS. Most servers run **Linux** — a free, rock-solid operating system built for exactly this: running quietly, for a long time, without a screen attached. WolfStack runs on Linux and manages a Linux machine for you.

You do **not** need to be a Linux expert to use WolfStack. That's the entire point of it — it gives you buttons for the things you'd otherwise type commands to do. You'll meet a command line later (it's genuinely useful), but you can do a huge amount without ever opening one.

## "Self-hosting" — running your own stuff

You already use servers all day: every website and app you open lives on someone else's server. **Self-hosting** means running those things on a server *you* control instead of renting them from a big company. Your own photo store instead of Google Photos. Your own notes app. Your own game server.

Why bother? **Control, privacy, cost, and learning.** Your data stays yours, you're not paying a monthly subscription forever, and you understand how it all fits together. That last one is why this is such a great thing to learn.

## A quick word on "containers"

You'll hear the word **container** a lot in this course. Don't worry about the detail yet — for now, just hold this picture: a container is **a tidy box you put one app in** so it can't make a mess of the rest of the machine. It's how modern servers stay organised. We'll explain the types (Docker, LXC, VM) properly when you actually need them — never before.

## ✓ What you just learned

- A **server** is just an always-on computer whose job is to run things other devices connect to.
- Most servers run **Linux**, and WolfStack gives you buttons instead of commands — you don't need to be an expert.
- **Self-hosting** = running apps on a machine you control, for privacy, cost, control, and the joy of understanding it.
- A **container** is just "a tidy box for one app." More on that later, when it matters.
