What Is Global WolfNet?

WolfStack automatically sets up a WolfNet encrypted mesh network within each cluster. But WolfNet itself doesn't care about clusters â€” it's a pure network-layer mesh. That means you can extend any cluster's WolfNet to include your local machine, and from there reach every server across every cluster as if they were all on the same LAN.

The result is a single flat private network that spans your entire infrastructure. You SSH, ping, query databases, and access services using stable WolfNet IP addresses â€” regardless of where the physical servers are hosted, what their public IPs are, or whether they're behind NAT.

Why This Works

WolfNet uses Peer Exchange (PEX) â€” once your local machine peers with any one node in the mesh, every other node in the mesh automatically learns about your machine within ~30 seconds. You don't have to manually add all 17+ servers to your local config. You only need to peer with one reachable node per cluster as an entry point.

Step-by-Step Setup

Step 1 â€” Install WolfNet on your local machine

bash 
curl -sSL https://raw.githubusercontent.com/wolfsoftwaresystemsltd/WolfScale/main/wolfnet/setup.sh | sudo bash

When prompted, choose a WolfNet IP that isn't already in use on your servers (e.g. 10.0.10.100). You can leave gateway mode off.

Step 2 â€” Find each entry-point node's public key

On each cluster's entry-point server, run:

bash 
# Run this on node-a1, node-b1, node-c1, node-d1 etc.
wolfnet pubkey

# Also note its public IP:
curl -s ifconfig.me

Step 3 â€” Replace your local config

Edit /etc/wolfnet/config.toml on your local machine and fill in the public keys and IPs you collected above, using the template in the section above.

Step 4 â€” Add your local machine as a peer on each entry-point node

On each entry node (node-a1, node-b1, etc.) you need to tell WolfNet about your local machine. The easiest way is the invite/join flow:

bash 
# On the server (node-a1):
sudo wolfnet invite

# Copy the output command to your LOCAL machine and run it:
sudo wolfnet --config /etc/wolfnet/config.toml join eyJwa...

# Copy the reverse token back to the server and run it:
sudo wolfnet --config /etc/wolfnet/config.toml join eyJlc...

# Restart WolfNet on the server:
sudo systemctl restart wolfnet

# Repeat for node-b1, node-c1, node-d1 etc.

Alternatively, manually append a [[peers]] block for your local machine to each server's config, then restart WolfNet. Either method works.

Step 5 â€” Restart WolfNet on your local machine

bash 
sudo systemctl restart wolfnet

Step 6 â€” Verify

bash 
# See all peers discovered so far (PEX propagates within ~30s):
wolfnetctl peers

# Ping a server in Cluster A:
ping 10.0.10.1

# Ping a server in Cluster D:
ping 10.0.10.30

# SSH into any server by WolfNet IP:
ssh user@10.0.10.12
âœ… Within about 30 seconds of restarting, WolfNet's Peer Exchange will propagate your local machine's details to every node across all clusters â€” and you'll see all 17 servers appear in wolfnetctl peers automatically.

Tips & Tricks

Tip Details
Use /etc/hosts aliases Add entries like 10.0.10.1 node-a1 to your local /etc/hosts so you can ssh node-a1 instead of remembering IPs.
Only one entry point per cluster needed PEX automatically distributes peer info. You don't need to manually add all 17 servers â€” just one reachable node per cluster.
Dynamic home IP? Your local machine initiates outbound connections, so it doesn't need a static IP or port forwarding. The servers need reachable public endpoints.
Laptop on the go WolfNet handles endpoint roaming automatically. Switch from WiFi to mobile data and your connections recover within ~30 seconds.
Multiple local machines Give each device its own WolfNet IP (e.g. 10.0.10.100, 10.0.10.101, â€¦) and repeat the invite process.
Confirm WolfNet IPs on servers Run wolfnetctl status or ip addr show wolfnet0 on any server to see its WolfNet IP.
